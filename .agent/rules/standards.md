# Standards

These are cumulative goals refined from many refactors of this codebase.
Each time this code base is refactored it should be because something in this file changed.

## GitHub Workflows

- workflows should orchestrate, not execute
  - workflows may call out to external actions or internal scripts, but must not execute full steps by themselves
  - all "run" scripts should be placed in the .github/workflows/scripts directory
  - all "github-script" scripts should be placed in the .github/workflows/github-scripts directory
- all scripts should be validated in the pull_request.yaml workflow
- only pre-approved action namespaces are allowed
  - approved namespaces are communicated here: https://github.com/rancher/security-team/blob/main/docs/standards/rancher-gha-standards.md#allowed-github-actions
  - list of important action namespaces (from that list):

## Terraform

- use variable validations for simple validations
- use check blocks for complex validations
  - validations which include inputs from other variables must go here
- use terraform_data preconditions for complex validations that can't be in check blocks
  - validations which include outputs from other resources must go here

## Test Suite

- the ./test directory should be a stand-alone Go module named "test"
- the ./test module should have a package for each test
- the ./test structure should be as DRY as possible
  - if possible, extract the creation of fixtures into its own package
  - test packages should validate a specific fixture or validate a specific test on multiple fixtures
- the ./test module should establish a terraform plugin cache for each test
- the ./test plugin cache should be seeded from the global cache established in the run_tests.sh script

## Running Tests

- there should be a makefile in the repo which runs outside of the development environment context
  - the makefile should handle entering the nix environment
  - the makefile should verify if nix is installed and error if it isn't found
- there should be a makefile with the 'lint', 'test', 'build', and 'cleanup' actions
- the lint action should run the script identical to what is used in GitHub workflows
- the run_tests.sh should enable lint testing with the --lint-only option
- the test action should run the run_tests.sh script without any options
- the build action should build up the global plugin cache and validate our examples
- the run_tests.sh should enable building with the --build-only option
- the cleanup action should accept an id and run the cleanup.sh script with that id
- the run_tests.sh should enable cleanup with the --cleanup_id option
- acceptance tests should be run using the run_tests.sh script
- a separate script should exist to clean up resources generated by a test run named cleanup.sh
- run_tests should have a speed option which sets the number of consecutive tests and test packages
- run_tests should have a "slow-mode" option which sets the speed to 1
- run_tests should have the option to set the test package, or fixture, or combination, or group of combinations
- run_tests should implement a global plugin cache for Terraform modules
- run_tests should have options to rerun failed tests and skip cleanup for debugging
- run_tests should have the option to run the cleanup script
- the cleanup script should execute in a trap within the run_tests script
  - so that even if the run_tests script panics or is canceled cleanup is still run
- the cleanup script should accept an optional id
- if not given an id the cleanup script should look for resources with the tag: "Owner" = "terraform-ci@suse.com"
